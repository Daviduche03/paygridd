# API Reference

Base URL: `https://api.paygrid.xyz/api/v1`

All endpoints require a valid API key sent via the `Authorization` header. Responses follow a uniform envelope:

```json
{
  "success": true,
  "data": { ... },
  "meta": { "cursor": "...", "total": 100 }
}
```

Errors return:

```json
{
  "success": false,
  "error": "Human-readable error message"
}
```

---

## Authentication

```bash
Authorization: Bearer pg_<your_api_key>
```

API keys are generated from the PayGrid dashboard under **Settings > Developer**. Each key can be scoped to a subset of resources. An empty scope list grants full access.

### Scopes

| Scope | Access |
|-------|--------|
| `customers.read` | List and view customers |
| `customers.write` | Create and update customers |
| `invoices.read` | List and view invoices |
| `invoices.write` | Create, update, cancel, and send invoices |
| `transactions.read` | List and view transactions |
| `business.read` | View business profile |
| `webhook.read` | List and view webhook endpoints |
| `webhook.write` | Create, update, and delete webhook endpoints |

### Rate Limiting

- **60 requests per minute** per IP across all `/api/v1` endpoints
- Exceeding the limit returns `429 Too Many Requests`

## Customers

Manage your customer directory.

### List Customers

```bash
GET /api/v1/customers
```

**Query Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `cursor` | string | Pagination cursor from a previous response `meta.cursor` |
| `pageSize` | integer | Results per page (default: 20, max: 100) |
| `q` | string | Search query — matches name, email, or phone |
| `sort` | string | Sort order: `name_asc`, `name_desc`, `created_at_asc`, `created_at_desc` |

**Scope:** `customers.read`

**Response**

```json
{
  "success": true,
  "data": [
    {
      "id": "uuid",
      "name": "Acme Corp",
      "email": "billing@acme.com",
      "phone": "+2348012345678",
      "billingEmail": "invoices@acme.com",
      "country": "Nigeria",
      "countryCode": "NG",
      "createdAt": "2026-06-01T10:00:00.000Z",
      "updatedAt": "2026-06-01T10:00:00.000Z"
    }
  ],
  "meta": {
    "cursor": "next_page_cursor",
    "total": 42
  }
}
```

```bash
POST /api/v1/customers
```

**Scope:** `customers.write`

**Request Body**

| `phone` | string | No | Phone number |
| `billingEmail` | string | No | Email for invoice delivery |
| `country` | string | No | Country name |
| `countryCode` | string | No | ISO 3166-1 alpha-2 country code |

**Response** `201 Created`

```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "name": "Acme Corp",
    "email": "billing@acme.com",
    "phone": null,
    "billingEmail": null,

    "countryCode": null,
    "createdAt": "2026-07-04T12:00:00.000Z",
    "updatedAt": "2026-07-04T12:00:00.000Z"
  }
}
```


**Scope:** `customers.read`

**Path Parameters**

| Param | Type | Description |
|-------|------|-------------|

## Invoices

Create, send, and manage invoices.

### Invoice Statuses

| Status | Description |
|--------|-------------|
| `draft` | Invoice being edited; not yet visible to customer |
| `scheduled` | Will be sent on a future date |
| `unpaid` | Sent to customer; awaiting payment |
| `overdue` | Past due date without payment |
| `paid` | Fully paid |
| `canceled` | Voided before or after sending |
| `refunded` | Payment returned to customer |

### List Invoices

```bash
GET /api/v1/invoices
```

**Scope:** `invoices.read`

**Query Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `cursor` | string | Pagination cursor |
| `pageSize` | integer | Results per page |
| `q` | string | Search by invoice number or customer name |
| `statuses` | string | Comma-separated list of statuses to filter |
| `customerIds` | string | Comma-separated list of customer UUIDs |
| `dueFilter` | string | `overdue` to show only past-due invoices |
| `sort` | string | Sort order: `issue_date_asc`, `issue_date_desc`, `due_date_asc`, `due_date_desc`, `amount_asc`, `amount_desc`, `created_at_asc`, `created_at_desc` |

**Response**

### Create Invoice

```bash
POST /api/v1/invoices
```

**Scope:** `invoices.write`

**Request Body**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `invoiceNumber` | string | **Yes** | Unique invoice number |
| `amount` | number | **Yes** | Invoice amount |
| `customerId` | uuid | No | Link to a customer |
| `virtualAccountId` | uuid | No | Link to a virtual account |
| `currency` | string | No | Currency (default: `NGN`) |
| `status` | string | No | Initial status (default: `draft`) |
| `issueDate` | string | No | ISO 8601 date |
| `dueDate` | string | No | ISO 8601 date |
| `lineItems` | array | No | Array of line item objects |

**Notes:**
- If `customerId` is provided, it must belong to your business.
- If `virtualAccountId` is provided, it must belong to your business.
- `amount` defaults to the sum of `lineItems` if not explicitly provided.

**Response** `201 Created`

```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "invoiceNumber": "INV-001",
    "amount": "150000.00",
    "currency": "NGN",
    "status": "draft",
    "paymentUrl": "https://paygrid.xyz/i/uuid",
    ...
  }
}
```

### Get Invoice

```bash
GET /api/v1/invoices/:id
```

**Scope:** `invoices.read`

**Path Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `id` | uuid | Invoice ID |

**Response** — Single invoice object with `paymentUrl`.

### Update Invoice

```bash
PUT /api/v1/invoices/:id
```

**Scope:** `invoices.write`

**Path Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `id` | uuid | Invoice ID |

**Request Body** — All fields optional.

| Field | Type | Description |
|-------|------|-------------|
| `amount` | number | Invoice amount |
| `currency` | string | Currency code |
| `customerId` | uuid | Customer UUID |
| `virtualAccountId` | uuid | Virtual account UUID |
| `issueDate` | string | ISO 8601 date |
| `dueDate` | string | ISO 8601 date |
| `lineItems` | array | Line items |

**Note:** The `status` field cannot be set to `paid` via this endpoint. Use the payment flow (customer pays via the payment URL) to mark invoices as paid.

### Cancel Invoice

```bash
POST /api/v1/invoices/:id/cancel
```

**Scope:** `invoices.write`

**Path Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `id` | uuid | Invoice ID |

**Constraints:**
- Cannot cancel an invoice with status `paid`, `canceled`, or `refunded`.

**Response** — Returns the updated invoice with status `canceled`.

### Send Invoice

```bash
POST /api/v1/invoices/:id/send
```

**Scope:** `invoices.write`

**Path Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `id` | uuid | Invoice ID |

**Constraints:**
- Only `draft` invoices can be sent.
- Sending transitions the invoice to `unpaid` and sets `issueDate` to the current date.

**Response** — Returns the updated invoice with status `unpaid`.

## Transactions

Incoming payments and financial events recorded against your business.

### Transaction Statuses

| Status | Description |
|--------|-------------|
| `pending` | Payment initiated, awaiting confirmation |
| `posted` | Payment confirmed and posted |
| `failed` | Payment attempt failed |
| `reversed` | Payment reversed to sender |

### Reconciliation Statuses

| Status | Description |
|--------|-------------|
| `pending` | Awaiting reconciliation |
| `matched` | Successfully matched to an invoice |
| `underpaid` | Amount received is less than the invoice amount |
| `overpaid` | Amount received exceeds the invoice amount |
| `duplicate` | Duplicate payment detected |
| `needs_review` | Requires manual review |

### List Transactions

```bash
GET /api/v1/transactions
```

**Scope:** `transactions.read`

**Query Parameters**

| Param | Type | Description |
|-------|------|-------------|
| `cursor` | string | Pagination cursor |
| `pageSize` | integer | Results per page |
| `q` | string | Search by sender name, narration, or reference |
| `status` | string | Filter by status: `pending`, `posted`, `failed`, `reversed` |
| `type` | string | Filter by type: `credit`, `debit` |
| `customerId` | uuid | Filter by customer |
| `virtualAccountId` | uuid | Filter by virtual account |
| `reconciliationStatus` | string | Filter by reconciliation status |
| `dateFrom` | string | Include transactions on or after this ISO 8601 date |
| `amountMin` | number | Minimum amount (inclusive) |
| `amountMax` | number | Maximum amount (inclusive) |

**Response**

```json
{
  "success": true,
  "data": [
    {

      "customerId": "uuid or null",

      "amount": "50000.00",
      "currency": "NGN",
      "status": "posted",
      "reconciliationStatus": "matched",
      "senderName": "John Doe",
      "senderBank": "GTBank",
      "narration": "Payment for INV-001",
      "occurredAt": "2026-07-04T10:30:00.000Z",

  ],
  "meta": { "cursor": "...", "total": 200 }

```

### Get Transaction

**Scope:** `transactions.read`

**Path Parameters**

```bash
GET /api/v1/business/balance
```

**Scope:** `business.read`

```json

  "success": true,
```

## Pagination

All list endpoints support cursor-based pagination. The response `meta` object contains a `cursor` field. Pass this value as the `cursor` query parameter in the next request to fetch the next page. When `cursor` is absent from the response, you have reached the last page.

```bash
GET /api/v1/customers?pageSize=50&cursor=eyJpZCI6IjEyMzQ1Njc4OTAi...
```

## Error Codes

| HTTP Code | Meaning |
|-----------|---------|
| `200` | Success |
| `201` | Created |
| `400` | Bad request — invalid input or business validation failure |
| `401` | Unauthorized — missing or invalid API key |
| `403` | Forbidden — API key lacks the required scope |
| `404` | Not found — resource does not exist |
| `429` | Too many requests — rate limit exceeded |
| `500` | Internal server error |

All errors return:

```json
{
  "success": false,
  "error": "Human-readable description of the problem"
}
```
